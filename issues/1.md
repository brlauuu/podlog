# Improve Telegram notification reporting for episode processing metrics

Currently, the Telegram notifications sent when an episode finishes processing lack detailed metrics that would help estimate queue processing time more accurately.

## Desired Improvements

### 1. Include Average Episode Length
Add the average episode length to the processing time calculations to provide better context for processing duration.

### 2. Calculate Processing Time Per Minute of Content
Calculate and display the average processing time per minute of content. For example:
- If a podcast is 60 minutes long and processing takes 90 minutes total, the processing factor is 1.5x
- This means each minute of content requires 1.5 minutes of processing time

### 3. Estimate Total Queue Processing Time
For all remaining episodes in the queue:
- Calculate the total minutes of content
- Apply the processing factor to estimate total processing time needed
- Display the factor prominently (e.g., "1.5x")

### Example Notification Format
The notification should include:
- Average episode length
- Processing factor (e.g., 1.5x)
- Estimated total processing time for remaining queue
- Clear breakdown so the user understands how these metrics relate
